# Актуальный контекст

## Текущий фокус
На момент анализа проект представляет собой полностью функциональное веб-приложение для прототипирования, включающее:

- Полноценную систему аутентификации (регистрация, логин, session management)
- REST API для работы с данными (CRUD operations)
- SPA интерфейс на Vue.js с админской панелью
- Миграционную систему БД
- Тестовые данные для демонстрации

## Недавние изменения (из .git истории)
Проект был создан как образовательный пример и включает:
- Основную структуру MVC
- Базу данных SQLite с миграциями
- Конфигурацию через environment variables
- Статические HTML страницы с Vue.js компонентами

## Активные решения
- Использование Bottle вместо более тяжелых фреймворков (Django/FastAPI) для простоты
- Отказ от ORM в пользу raw SQL для наглядности
- MD5 хэширование (небезопасно, но понятно для beginners)
- Подписанные cookies вместо JWT для упрощения
- Vue.js 3 Options API для совместимости и простоты

## Важные паттерны
- **Конфигурация через env vars**: Все настройки читаются из среды, что упрощает развертывание
- **Миграции**: JSON-based миграции позволяют эволюционировать схему БД
- **Инъекция зависимостей**: Bottle plugin автоматически предоставляет db connection
- **Компонентный фронтенд**: Vue.js компоненты организованы в отдельной папке

## Контекст разработки
- **Язык**: Python 3.x
- **Frontend**: Vue.js 3 + Vue Router 4 + Bootstrap 5
- **Database**: SQLite3
- **DevOps**: ngrok для публикации, venv для изоляции
- **Тестирование**: Импорт тестовых данных через test/import_test_data.py

## Что работает
- ✅ Регистрация новых пользователей
- ✅ Логин с созданием сессии (cookie)
- ✅ Разграничение админ/юзер
- ✅ CRUD операции с вещами (things)
- ✅ Миграции базы данных
- ✅ Запуск через Bottle сервер
- ✅ Статические файлы и SPA маршрутизация

## Известные ограничения
- MD5 небезопасен для production
- Нет HTTPS (требуется reverse proxy)
- SQLite не подходит для высокой конкуренции
- Cookies читаемы (не шифрованы)
- Нет валидации на frontend (только backend)
